Objective 
Digitize the provided paper-based data collection tool into a clean, user-friendly web application with 4 forms (Screening, Enrolment, Delivery, Closeout). The system must enforce workflow (Screening - Enrolment - Delivery), use Screening ID as the unique identifier, support user login, save data with user initials, store data in separate databases/tables. 
 
The focus of this assignment is on: 
вҖў	Practical workflow implementation  
вҖў	Data validation  
вҖў	User access control  
вҖў	Clean and functional system design  
The forms do not need to contain every question from the paper tools; only a representative subset is sufficient to demonstrate the system functionality. 
 
Login System (Password Protected) 
The system should include a simple secure login page requiring: 
вҖў	Username  
вҖў	Password  
After login: 
вҖў	Display the logged-in userвҖҷs initials (e.g., вҖңPAвҖқ for Patrobas Agwata)  вҖў 	Save all submitted records with:  вҖў User initials  
вҖў	Date and time stamp 
 
User Access Levels 
The system should support at least two user roles: 
1.	Data Manager -Permissions: View records  Enter new data  and Edit existing data  
2.	Field Technician- Permissions: View records only  
         	 	 	 Restrictions: Cannot add or edit data  
Have the following forms  
вҖў	Form 1: Screening 
вҖў	Form 2: Enrolment (Only accessible if Screening passed and Screening ID exists) 
вҖў	Form 3: Delivery (Only accessible if Enrolment record exists for the Screening ID) 
вҖў	Form 4: Closeout (Simple form for participants who drop out or are lost to follow-up) 
Validation 
вҖў	Screening ID is the unique key across all forms. 
вҖў	Enrolment form should show a dropdown/search of eligible Screening IDs (Study Eligibility = Yes and consent = Yes). 
вҖў	Delivery form should show a dropdown of enrolled Screening IDs. 
вҖў	Closeout can be done on any Screening ID. 
Other additional requirements 
вҖў	Have the system display summary of Screening/ Enrolment/ Delivery numbers either as raw numbers or in form of a table per site
